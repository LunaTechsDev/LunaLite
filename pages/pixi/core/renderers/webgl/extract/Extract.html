API Documentation

class Extract

package pixi.core.renderers.webgl.extract

@:native("PIXI.extract.Extract")

The extract manager provides functionality to export content from the renderers.

An instance of this class is automatically created by default, and can be found at renderer.plugins.extract

Constructor

new(renderer:Renderer)

Creates WebGLExtract

Parameters:

{PIXI.Renderer}

renderer - A reference to the current renderer

Methods

base64(?target:EitherType<DisplayObject, RenderTexture>):String

Will return a a base64 encoded string of this target. It works by calling WebGLExtract.getCanvas and then running toDataURL on that.

Parameters:

{PIXI.DisplayObject

| PIXI.RenderTexture} target - A displayObject or renderTexture to convert. If left empty will use the main renderer

Returns:

{String} A base64 encoded string of the texture.

canvas(?target:EitherType<DisplayObject, RenderTexture>):CanvasElement

Creates a Canvas element, renders this target to it and then returns it.

Parameters:

{PIXI.DisplayObject

| PIXI.RenderTexture} target - A displayObject or renderTexture to convert. If left empty will use the main renderer

Returns:

{HTMLCanvasElement} A Canvas element with the texture rendered on.

pixels(?target:EitherType<DisplayObject, RenderTexture>):Uint8ClampedArray

Will return a one-dimensional array containing the pixel data of the entire texture in RGBA order, with integer values between 0 and 255 (included).

Parameters:

{PIXI.DisplayObject

| PIXI.RenderTexture} target - A displayObject or renderTexture to convert. If left empty will use the main renderer

Returns:

{Uint8ClampedArray} One-dimensional array containing the pixel data of the entire texture
