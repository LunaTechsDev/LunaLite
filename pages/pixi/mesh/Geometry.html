Constructor

new(?buffers:Array<Buffer>, ?attributes:Dynamic)

The Geometry represents a model. It consists of two components:

GeometryStyle - The structure of the model such as the attributes layout GeometryData - the data of the model - this consists of buffers. This can include anything from positions, uvs, normals, colors etc.

@class @memberof PIXI.mesh

Parameters:

buffers

{Array} An array of buffers. optional.

[attributes]

{Dynamic} of the geometry, optional structure of the attributes layout

addAttribute(id:String, ?buffer:Buffer, ?size:Int, ?normalized:Bool, ?type:Float, ?stride:Float, ?start:Int):Geometry

Adds an attribute to the geometry

Parameters:

id

{String} the name of the attribute (matching up to a shader)

buffer

{Buffer} the buffer that holds the data of the attribute . You can also provide an Array and a buffer will be created from it.

size

{Int} the size of the attribute. If you have 2 floats per vertex (eg position x and y) this would be 2

normalized

{Bool} should the data be normalized.

type

{Float} what type of number is the attribute. Check {PIXI.TYPES} to see the ones available

stride

{Float} How far apart (in floats) the start of each value is. (used for interleaving data)

star

{Int} How far into the array to start reading values (used for interleaving data)

Returns:

{Geometry} returns self, useful for chaining.
